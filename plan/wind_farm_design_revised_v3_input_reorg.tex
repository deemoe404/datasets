\subsection{运行状态、脏数据与实现边界}

风电 SCADA/PMU 的难点不只来自气动耦合，还来自停机、告警、通信异常、传感器卡死以及不同链路时延。当前 v1 实现并未单独引入一条独立的 $u^{ops}$ 输入分支，而是采用如下固定约定：
\begin{itemize}
  \item 本地与场级事件摘要（如 `$evt\_*$`、`$farm_evt\_$*`）直接作为观测通道进入 $o_{i,t}$ 与 $f_t$，用于表达停机、警告、信息类事件及其活跃强度。
  \item 若未来补充限功率、控制模式或更细粒度的运行状态标签，可在后续版本中再单独扩展输入契约；当前 v1 文档不再把这类信号当作已实现输入。
\end{itemize}

脏数据处理方面，原始数据清洗与模型内 mask-aware 处理必须同时存在：前者负责明显异常点、重复记录、非物理值与时间戳错位的修正；后者负责在保留缺失模式信息的前提下完成状态估计。模型侧不允许把“已经插补后的值”误当作“真实观测”，因此插补、mask、time-gap 需要成组输入网络。

\section{适用于多风机风场风功率预测任务的时空序列预测网络}

该模型的核心思想是：在时刻 $t$ 利用真实观测对当前隐状态进行估计，并以该隐状态为起点 rollout 得到时刻 $t+1$ 的先验隐状态；随后再结合时刻 $t+1$ 的真实观测对该先验隐状态进行修正，从而得到更准确的后验隐状态，并为后续 rollout 提供更可靠的状态基础。换言之，模型通过“rollout $\rightarrow$ 观测修正”的交替过程，不断提升隐状态估计质量，以改善后续预测结果。

在推理阶段，采用默认配置 $\Delta=10\,\mathrm{min}$、lookback 长度为 $144$ 步（对应 $24$ 小时）、horizon 长度为 $36$ 步（对应 $6$ 小时）。由于模型每次仅接收单个时间刻的输入，因此首先需要利用 lookback window 对隐状态进行预热。在预热阶段，模型在每一个历史时刻仅执行一步 rollout，然后利用下一时刻的真实观测对 rollout 得到的先验隐状态进行更新；这一过程沿历史窗口持续进行，直至处理完整个 lookback window。完成预热后，模型再从时刻 $t$ 的后验隐状态出发，连续执行 $36$ 步 rollout，从而生成未来 $6$ 小时的预测结果。因此，从功能上看，历史窗口对应的是状态估计 / 同化（filtering / assimilation）过程，而未来窗口对应的是开环隐状态传播（open-loop rollout）过程。

\subsection{结构详情}

\subsubsection{初始化}

\]

其中：
\[

\]

\subsubsection{本地观测编码}
\[
r_{i,t}=\mathrm{Enc}_{obs}\left([o_{i,t},\, q^{obs}_{i,t},\, \Delta t^{obs}_{i,t},\, a^{audit}_{i,t},\, s_i]\right).
\]

\subsubsection{图消息传递与 wake 动态}
当前时刻的 update-stage 源机组摘要写为
\[
v^{src}_{j,t}=\mathrm{Enc}^{upd}_{src}\left([o_{j,t},\, q^{obs}_{j,t},\, \Delta t^{obs}_{j,t},\, a^{audit}_{j,t},\, z^-_{j,t},\, h^-_{j,t},\, s_j]\right).
\]

若原始 wake geometry 为边 $j\to i$ 提供二维坐标差与 rotor-diameter 距离
\[
(\Delta x_{ij},\, \Delta y_{ij},\, d^{(RD)}_{ij}),
\]
则当前风向下的动态 wake 特征定义为
\[
d^{\parallel}_{ij,t}=\frac{\Delta x_{ij} C_t+\Delta y_{ij} S_t}{d^{(RD)}_{ij}},
\qquad
d^{\perp}_{ij,t}=\frac{-\Delta x_{ij} S_t+\Delta y_{ij} C_t}{d^{(RD)}_{ij}},
\]
\[
\psi^{wake}_{ij,t}=\sigma\!\left(\kappa d^{\parallel}_{ij,t}\right)\cdot
\exp\!\left(-\frac{\mathrm{ReLU}(d^{\parallel}_{ij,t})}{\lambda_x}\right)\cdot
\exp\!\left(-\left(\frac{d^{\perp}_{ij,t}}{\lambda_y}\right)^2\right),
\]
\[
p^{dyn,upd}_{i\leftarrow j,t}=\left[d^{\parallel}_{ij,t},\, d^{\perp}_{ij,t},\, \psi^{wake}_{ij,t}\right].
\]

于是 update-stage 的边消息与门控写为
\[
e_{i\leftarrow j,t}^{upd}=\mathrm{MLP}_e^{upd}\left([s_i,\, s_j,\, p_{i\leftarrow j}^{stat},\, p_{i\leftarrow j,t}^{dyn,upd},\, v^{src}_{j,t},\, g_t^-,\, s^{site}_t,\, \mathrm{cal}_t]\right),
\]
\[
a_{i\leftarrow j,t}^{upd}=\sigma\left(\mathrm{MLP}_a^{upd}\left([s_i,\, s_j,\, p_{i\leftarrow j}^{stat},\, p_{i\leftarrow j,t}^{dyn,upd},\, v^{src}_{j,t},\, g_t^-,\, s^{site}_t,\, \mathrm{cal}_t]\right)\right),
\]
\[
m_{i,t}^{upd}=\frac{\sum_{j\neq i} a^{upd}_{i\leftarrow j,t}e^{upd}_{i\leftarrow j,t}}{\epsilon+\sum_{j\neq i} a^{upd}_{i\leftarrow j,t}}.
\]

\subsubsection{共享的 pred-transition kernel}
历史窗中的 prior propagation 与未来窗中的 open-loop rollout 共用同一套
Transition / Transition$_g$ 参数。给定某一步 transition 之前的节点/全局状态
\[
\left\{z_i,\, h_i\right\}_{i=1}^N,\qquad g,
\]
以及该步目标时刻的日历编码 $\mathrm{cal}_{next}$，先由全局状态解码下一步站点级摘要
\[
\hat c^{met}_{next}=\mathrm{Dec}^{met}_g\left(g,\, \mathrm{cal}_{next}\right),\qquad
\hat c_{next}=\left[\hat c^{met}_{next},\, \mathrm{cal}_{next}\right].
\]
记
\[
\hat c^{met}_{next}=
\left[
  \hat{\bar{ws}}_{next},\,
  \hat C_{next},\,
  \hat S_{next},\,
  \hat R_{next},\,
  \hat\rho^{ws}_{next},\,
  \hat\rho^{wd}_{next}
\right],
\]
再由当前节点状态构造 source summary
\[
\hat\eta^{src}_{j,next}=\mathrm{Enc}^{pred}_{src}\left([z_j,\, h_j,\, g,\, s_j]\right).
\]
对应的预测期 wake 动态写为
\[
\hat d^{\parallel}_{ij,next}=\frac{\Delta x_{ij}\hat C_{next}+\Delta y_{ij}\hat S_{next}}{d^{(RD)}_{ij}},
\qquad
\hat d^{\perp}_{ij,next}=\frac{-\Delta x_{ij}\hat S_{next}+\Delta y_{ij}\hat C_{next}}{d^{(RD)}_{ij}},
\]
\[
\hat\psi^{wake}_{ij,next}=\sigma\!\left(\kappa \hat d^{\parallel}_{ij,next}\right)\cdot
\exp\!\left(-\frac{\mathrm{ReLU}(\hat d^{\parallel}_{ij,next})}{\lambda_x}\right)\cdot
\exp\!\left(-\left(\frac{\hat d^{\perp}_{ij,next}}{\lambda_y}\right)^2\right),
\]
\[
\hat p^{dyn,pred}_{i\leftarrow j,next}=\left[\hat d^{\parallel}_{ij,next},\, \hat d^{\perp}_{ij,next},\, \hat\psi^{wake}_{ij,next}\right].
\]

由此得到 pred-stage 的图消息
\[
e^{pred}_{i\leftarrow j,next}=\mathrm{MLP}_e^{pred}\left([s_i,\, s_j,\, p^{stat}_{i\leftarrow j},\, \hat p^{dyn,pred}_{i\leftarrow j,next},\, \hat\eta^{src}_{j,next},\, g,\, \hat c_{next}]\right),
\]
\[
a^{pred}_{i\leftarrow j,next}=\sigma\left(\mathrm{MLP}_a^{pred}\left([s_i,\, s_j,\, p^{stat}_{i\leftarrow j},\, \hat p^{dyn,pred}_{i\leftarrow j,next},\, \hat\eta^{src}_{j,next},\, g,\, \hat c_{next}]\right)\right),
\]
\[
m^{pred}_{i,next}=\frac{\sum_{j\neq i} a^{pred}_{i\leftarrow j,next}e^{pred}_{i\leftarrow j,next}}{\epsilon+\sum_{j\neq i} a^{pred}_{i\leftarrow j,next}}.
\]

随后用共享的 local/global transition 推进一步状态：
\[
z'_i=\mathrm{GRU}^{tr}_z\left([m^{pred}_{i,next},\, g,\, \hat c^{met}_{next},\, \mathrm{cal}_{next},\, h_i,\, s_i],\, z_i\right),
\]
\[
h'_i=\mathrm{GRU}^{tr}_h\left([z'_i,\, m^{pred}_{i,next},\, g,\, \hat c^{met}_{next},\, \mathrm{cal}_{next},\, s_i],\, h_i\right),
\]
\[
\bar n^{tr}_{next}=\frac{1}{N}\sum_{i=1}^{N}[z'_i,\, h'_i],\qquad
g'=\mathrm{GRU}^{tr}_g\left([\bar n^{tr}_{next},\, \hat c^{met}_{next},\, \mathrm{cal}_{next}],\, g\right).
\]
\noindent 在历史 prior propagation 中，上述 generic 变量替换为
\[
(z_i,\, h_i,\, g,\, \mathrm{cal}_{next})=(z^+_{i,t-1},\, h^+_{i,t-1},\, g^+_{t-1},\, \mathrm{cal}_{t}),
\]
而在未来 rollout 中替换为
\[
(z_i,\, h_i,\, g,\, \mathrm{cal}_{next})=(z_{i,t+\tau-1|t},\, h_{i,t+\tau-1|t},\, g_{t+\tau-1|t},\, \mathrm{cal}_{t+\tau}).
\]

\subsubsection{共享的 posterior correction kernel}
给定时刻 $t$ 的 prior state
\[
\left\{z^-_{i,t},\, h^-_{i,t}\right\}_{i=1}^N,\qquad g_t^-,
\]
先由 prior state 解码当前观测：
\[
\hat o^-_{i,t}=\mathrm{Dec}^{prior}_{obs}\left([z^-_{i,t},\, h^-_{i,t},\, g_t^-,\, s_i]\right).
\]
再构造 masked residual 与 innovation：
\[
\delta o_{i,t}=q^{obs}_{i,t}\odot\left(o_{i,t}-\hat o^-_{i,t}\right),
\qquad
\nu_{i,t}=\mathrm{Enc}_{\nu}\left([\hat o^-_{i,t},\, \delta o_{i,t},\, q^{obs}_{i,t},\, \Delta t^{obs}_{i,t},\, r_{i,t}]\right).
\]
局部 posterior update 的候选状态写为
\[
\tilde z^+_{i,t}=\mathrm{GRU}^{upd}_z\left([\nu_{i,t},\, m^{upd}_{i,t},\, g_t^-,\, c_t,\, s_i],\, z^-_{i,t}\right),
\]
\[
\tilde h^+_{i,t}=\mathrm{GRU}^{upd}_h\left([\tilde z^+_{i,t},\, \nu_{i,t},\, m^{upd}_{i,t},\, g_t^-,\, c_t,\, s_i],\, h^-_{i,t}\right).
\]
记
\[
\rho^{node}_{i,t}=
\begin{cases}
1, & \text{节点 } i \text{ 在时刻 } t \text{ 至少存在一个可用本地观测},\\
0, & \text{否则},
\end{cases}
\]
则实际 posterior state 写为
\[
z^+_{i,t}=\rho^{node}_{i,t}\tilde z^+_{i,t}+\left(1-\rho^{node}_{i,t}\right)z^-_{i,t},
\]
\[
h^+_{i,t}=\rho^{node}_{i,t}\tilde h^+_{i,t}+\left(1-\rho^{node}_{i,t}\right)h^-_{i,t}.
\]
随后对节点 posterior state 做池化并更新全局状态：
\[
\bar n_t^+=\frac{\sum_i \rho^{node}_{i,t}\,[z^+_{i,t},\, h^+_{i,t}]}{\epsilon+\sum_i \rho^{node}_{i,t}},\qquad
g_t^+=\mathrm{GRU}^{upd}_g\left([\bar n_t^+,\, c_t],\, g_t^-\right).
\]
历史期 posterior 重建则由
\[
\hat o^+_{i,t}=\mathrm{Dec}^{post}_{obs}\left([z^+_{i,t},\, h^+_{i,t},\, g_t^+,\, s_i]\right)
\]
给出。

\subsubsection{历史窗口控制流}
历史窗口中的状态闭环按 ``\textbf{先 Transition 得到 prior，再用真实观测做 Update}'' 执行。
第一个历史步 $t=t_0$ 没有上一时刻的历史后验，因此直接从初始化状态
\[
\left\{z^+_{i,t_0-1},\, h^+_{i,t_0-1}\right\}_{i=1}^N,\qquad g^+_{t_0-1}
\]
作为第一个 history step 的 prior，即
\[
z^-_{i,t_0}=z^+_{i,t_0-1},\qquad h^-_{i,t_0}=h^+_{i,t_0-1},\qquad g^-_{t_0}=g^+_{t_0-1},
\]
再进入上文定义的 posterior correction kernel。对每个后续历史步 $t>t_0$，先以上一时刻后验
\[
\left\{z^+_{i,t-1},\, h^+_{i,t-1}\right\}_{i=1}^N,\qquad g^+_{t-1}
\]
调用共享的 pred-transition kernel，并取
\[
z^-_{i,t}=z'_i,\qquad h^-_{i,t}=h'_i,\qquad g_t^-=g',
\]
再结合当前真实观测 $\{o_{i,t}\}_{i=1}^N$ 与共享上下文 $c_t$ 调用上文的 posterior correction kernel，得到
\[
\left\{z^+_{i,t},\, h^+_{i,t}\right\}_{i=1}^N,\qquad g_t^+.
\]

\subsubsection{未来 rollout 控制流}
未来窗不再接入真实观测，只保留 open-loop transition。以历史窗口末端后验
\[
\left\{z^+_{i,t},\, h^+_{i,t}\right\}_{i=1}^N,\qquad g_t^+
\]
为起点，对每个 horizon 步 $\tau=1,\ldots,H$ 反复调用共享的 pred-transition kernel，并取
\[
z_{i,t+\tau|t}=z'_i,\qquad h_{i,t+\tau|t}=h'_i,\qquad g_{t+\tau|t}=g'.
\]
每一步 transition 同时给出内部预测的站点级摘要 $\hat c^{met}_{t+\tau}$。训练阶段从真实未来观测构造站点级统计量，并记为
\[
c^{met,*}_{t+\tau}=\left[\widetilde{\bar{ws}}_{t+\tau},\, C_{t+\tau},\, S_{t+\tau},\, R_{t+\tau},\, \rho^{ws}_{t+\tau},\, \rho^{wd}_{t+\tau}\right]
\]
并将其作为 $\hat c^{met}_{t+\tau}$ 的弱监督目标；pred-transition kernel 中的
source summary $\hat\eta^{src}_{j,next}$ 仅作为 latent source summary，通过 forecast / graph losses 间接训练。

\subsubsection{缺测与池化规则}
时间缺口特征 $\Delta t^{obs}_{i,t}$ 由最近一次可用观测距离按时间步累计得到，并在
\[
\Delta t_{max}=72\Delta=720\,\mathrm{min}
\]
处截断。若某节点在时刻 $t$ 的全部本地观测通道都不可用，则由 $\rho^{node}_{i,t}=0$ 触发
\[
z^+_{i,t}=z^-_{i,t},\qquad h^+_{i,t}=h^-_{i,t},
\]
即跳过该节点的 posterior correction。若当前时刻所有节点都没有可用本地观测，则
$\bar n_t^+$ 不再使用掩码平均，而退回到所有节点 posterior state 的简单平均。

\subsubsection{Decode 与输出约束}
\[
\hat y^{farm,pu}_{t+\tau|t}
=
b_0+b_1\cdot \frac{1}{N}\sum_{i=1}^N \hat y^{pu}_{i,t+\tau|t},
\qquad b_0\leftarrow 0,\; b_1\leftarrow 1
\]

其中：

\paragraph{bounded absolute}
\[
\hat y^{raw}_{i,t+\tau|t}
=
\mathrm{MLP}^{raw}_{dec}\left(
[h_{i,t+\tau|t},\, z_{i,t+\tau|t},\, g_{t+\tau|t},\, \mathrm{emb}(\tau),\, s_i]
\right)
\]
\[
\hat y^{pu}_{i,t+\tau|t}
=
(1+\epsilon_y)\,\sigma\left(\hat y^{raw}_{i,t+\tau|t}\right),
\qquad \epsilon_y=0.05
\]


\paragraph{residual-over-persistence (\texttt{residual\_persistence})}
\[
\hat y^{raw}_{i,t+\tau|t}
=
\mathrm{MLP}^{raw}_{dec}\left(
[h_{i,t+\tau|t},\, z_{i,t+\tau|t},\, g_{t+\tau|t},\, \mathrm{emb}(\tau),\, s_i]
\right)
\]
\[
y^{anchor}_{i,t}
=
\text{最后一个有效历史 } y^{pu}_{i,\cdot}
\]
\[
\hat y^{pu}_{i,t+\tau|t}
=
\mathrm{clip}\!\left(
y^{anchor}_{i,t}+\hat y^{raw}_{i,t+\tau|t},
\,0,\,
1+\epsilon_y
\right),
\qquad \epsilon_y=0.05
\]


\paragraph{当前实验状态（2026-04-21）}
基于 scratch 队列 \texttt{residual\_head\_plan\_20260419\_farm0p05} 的完成结果，\texttt{residual\_persistence} 现在更适合作为主线 family 的 experimental working head，而不是立即替换 canonical 默认头。在 \texttt{farm\_loss\_weight=0.05}、按 \texttt{val\_rmse\_pu} 选 best epoch 的 repro 中，residual 相对 bounded absolute 在 val/test rolling overall 上都更好（val RMSE 0.16297 vs 0.16548，test RMSE 0.16903 vs 0.17399，MAE 也同步下降），说明这次收益不只是 test-side 偶然波动。进一步地，在 \texttt{farm\_loss\_weight=0.0} 的双 contract 最小复跑里，residual 在 \texttt{val\_rmse\_pu} 与 \texttt{val\_mae\_pu} 两种 selection metric 下都优于 absolute；但 carry-over scratch eval 并没有带来增益，canonical 与 residual 在 \texttt{rolling\_origin\_carry\_over} 下都比 \texttt{rolling\_origin\_no\_refit} 略差。因此当前更稳的研究定位是：把 residual 视为 decoder-side 的稳定改进并带入后续原型线，而不要把这轮结果解读成“online state persistence 已经被证实有效”。

\paragraph{global + local decoder follow-ups（2026-04-22 代码版）}
在当前 trunk 上，后续 decoder-side 结构探索先收敛成两个不引入 cluster state 的 whole-farm 版本。第一条是 \texttt{global\_local\_residual}：把 anchor-relative 预测拆成一个只依赖 \(g_{t+\tau|t}\) 与 \(\mathrm{emb}(\tau)\) 的全场共享 residual，再加一个只依赖 \([h_{i,t+\tau|t}, z_{i,t+\tau|t}, s_i, \mathrm{emb}(\tau)]\) 的 local correction；local 分支在每个 horizon 上都做按 turbine weight 的零均值约束，从而避免 global/local 学同一件事。第二条是 \texttt{global\_local\_increment}：保持同样的 global/local 分解和 zero-mean 约束，但把输出语义改成 per-step increment，并从 persistence anchor 逐步累积回 level。当前这两条线只完成代码、测试与文档对齐；实验尚未启动，因此还不能对哪条语义更优下结论。

\paragraph{loss-side 更新（2026-04-22）}
derived-ramp auxiliary 的第一轮最小 ablation 只在当前 \texttt{residual\_persistence + farm\_loss\_weight=0.0} trunk 上验证；这条旧线现在已经收口，不再继续追加单尺度 \(K=1\) ramp。原因有两点：第一，已完成的训练结果只给出极小且不够干净的 level-side 变化，尚不足以支持把单尺度 ramp 当成主杠杆；第二，旧的 saved-checkpoint scratch eval 路径在 \texttt{rolling\_origin\_carry\_over} 上 wall-clock 成本过高，\texttt{ramp0p02} 的 eval 运行超过 8 小时仍未产出 summary/diagnostics，因此被主动放弃，\texttt{ramp0p05} 也在启动前取消。主线因此切到 \texttt{ramp v2}：固定多尺度一阶差分 \(K\in\{3,6\}\)，同时把 \texttt{rolling\_origin\_no\_refit} 的 checkpoint scratch eval 改成 batched path；而 \texttt{persistent\_chunk6} 继续保持独立结构线，第一版仍固定 \texttt{ramp\_loss\_weight=0.0}，只有当新的 ramp 设计先在当前 residual trunk 上证明自己，才考虑接到 chunk6 上。

\section{可直接启动的超参数与训练策略}

v1 将所有\textbf{连续回归头}统一固定为 Huber loss；二值状态位只作为输入，不在 v1 中单独监督。记
\[
\mathrm{Huber}_{\delta}(u)=
\begin{cases}
\frac{1}{2}u^2/\delta, & |u|\le \delta,\\
|u|-\frac{1}{2}\delta, & |u|>\delta.
\end{cases}
\]
在默认的 $L=144$、$H=36$、$\Delta=10\,\mathrm{min}$ 配置下，固定 burn-in 长度 $L_{burn}=108$，因此历史重建只作用于最后 $36$ 个历史时间步：
\[
\mathcal T_{hist}^{loss}=\{t_0+108,\ldots,t_0+143\},\qquad w_\tau=\frac{1}{H},\ \tau=1,\ldots,H.
\]
其中 $w_\tau$ 为未来 horizon 的固定均匀权重，v1 不再保留“是否加权”的自由度。

首先定义主预测损失：
\[
L_{\mathrm{forecast}}
=
\frac{1}{NH}\sum_{i=1}^{N}\sum_{\tau=1}^{H} q^{y}_{i,t+\tau}\,w_\tau\,\mathrm{Huber}_{\delta_y}\!\left(\hat y^{pu}_{i,t+\tau|t}-y^{pu}_{i,t+\tau}\right),
\qquad \delta_y=0.03.
\]

再定义 posterior decode：
\[
\hat o^+_{i,t}=\mathrm{Dec}^{post}_{obs}\left([z^+_{i,t},\, h^+_{i,t},\, g_t^+,\, s_i]\right).
\]
于是历史重建损失固定为
\[
\begin{aligned}
L_{\mathrm{hist\_recon}}
&=
\frac{1}{N|\mathcal T_{hist}^{loss}|}
\sum_{t\in \mathcal T_{hist}^{loss}}\sum_{i=1}^{N}
\Big(
0.25\,\mathrm{Huber}_{\delta_o}\big(q^{obs}_{i,t}\odot(\hat o^-_{i,t}-o_{i,t})\big)\\
&\hspace{3.8cm}+
0.75\,\mathrm{Huber}_{\delta_o}\big(q^{obs}_{i,t}\odot(\hat o^+_{i,t}-o_{i,t})\big)
\Big),
\qquad \delta_o=0.05.
\end{aligned}
\]

场级一致性损失固定在 farm per-unit 口径上定义。记
\[
y^{farm,pu}_{t}=\frac{P^{farm}_{t}}{\sum_i rated\_power_i},
\qquad
\hat y^{farm,pu}_{t+\tau|t}=\frac{\hat P^{farm}_{t+\tau|t}}{\sum_i rated\_power_i},
\]
则
\[
L_{\mathrm{farm}}
=
\frac{1}{H}\sum_{\tau=1}^{H}
\mathrm{Huber}_{\delta_f}\left(\hat y^{farm,pu}_{t+\tau|t}-y^{farm,pu}_{t+\tau}\right),
\qquad \delta_f=0.03.
\]

为避免 ``$\hat c^{met}$ 被赋予物理名字但训练上毫无约束''，v1 额外引入一个弱监督的站点级气象摘要损失。将
\[
\hat c^{met}_{t+\tau}=
\left[\widehat{\bar{ws}}_{t+\tau},\, \hat C_{t+\tau},\, \hat S_{t+\tau},\, \hat R_{t+\tau},\, \hat\rho^{ws}_{t+\tau},\, \hat\rho^{wd}_{t+\tau}\right]
\]
与真实未来统计量
\[
c^{met,*}_{t+\tau}=
\left[\bar{ws}_{t+\tau},\, C_{t+\tau},\, S_{t+\tau},\, R_{t+\tau},\, \rho^{ws}_{t+\tau},\, \rho^{wd}_{t+\tau}\right]
\]
对齐，定义
\[
\begin{aligned}
L_{\mathrm{met}}
&=
\frac{1}{H}\sum_{\tau=1}^{H}
\Big(
\mathrm{Huber}_{0.05}(\widehat{\bar{ws}}_{t+\tau}-\bar{ws}_{t+\tau})
+0.5\,\|\hat C_{t+\tau}-C_{t+\tau}\|_2^2
+0.5\,\|\hat S_{t+\tau}-S_{t+\tau}\|_2^2 \\
&\hspace{2.2cm}+
0.25\,\mathrm{Huber}_{0.05}(\hat R_{t+\tau}-R_{t+\tau})
+0.25\,\mathrm{Huber}_{0.05}(\hat\rho^{ws}_{t+\tau}-\rho^{ws}_{t+\tau})
+0.25\,\mathrm{Huber}_{0.05}(\hat\rho^{wd}_{t+\tau}-\rho^{wd}_{t+\tau})
\Big).
\end{aligned}
\]
$\hat\eta^{src}$ 在 v1 中仍被视为 latent source summary，默认不加入逐维直接监督，因此固定取 $\lambda_{src}=0$，不启用 $L_{src}^{opt}$。

Innovation 正则项也在 v1 中定版为
\[
L_{\mathrm{innovation}}
=
\frac{1}{N|\mathcal T_{hist}^{loss}|}
\sum_{t\in \mathcal T_{hist}^{loss}}\sum_{i=1}^{N}
\left(
\frac{\|z^+_{i,t}-z^-_{i,t}\|_2^2}{d_z}
+
0.5\,\frac{\|h^+_{i,t}-h^-_{i,t}\|_2^2}{d_h}
\right)
+
0.25\,\frac{1}{|\mathcal T_{hist}^{loss}|}
\sum_{t\in \mathcal T_{hist}^{loss}}
\frac{\|g_t^+-g_t^-\|_2^2}{d_g}.
\]

综合考虑预测精度、状态一致性、场级约束与未来站点摘要监督，v1 的最终训练目标固定为
\[
L
=
1.0\,L_{\mathrm{forecast}}
+0.2\,L_{\mathrm{hist\_recon}}
+0.1\,L_{\mathrm{farm}}
+0.05\,L_{\mathrm{met}}
+10^{-4}L_{\mathrm{innovation}}.
\]

此外，v1 默认\textbf{不将}上一时刻预测值 $\hat{y}_{t+\tau-1|t}$ 回灌给下一时刻 Transition 输入；Transition 只在 latent space 中运行。因此 scheduled sampling 在本版本中固定关闭，不作为标准训练组件。只有当后续实验显式改成基于输出回灌的自回归解码器时，才再考虑引入 scheduled sampling。

\subsection{评估与消融协议}
为形成完整的研究闭环，v1 在首轮实验中固定如下评估协议：
\begin{itemize}
  \item \textbf{切分方式：}仅允许按时间连续块切分 train / validation / test，禁止随机打散；若数据覆盖多个季节，至少额外报告一次 rolling-origin 的跨季节回测结果。
  \item \textbf{主指标：}报告 turbine-level 与 farm-level 两组指标，至少包含 MAE、RMSE；同时给出 per-unit 口径与 kW 口径；对未来 $36$ 步逐 horizon 报告，并汇总成短期（1--12 步）、中期（13--24 步）、长期（25--36 步）三档结果。
  \item \textbf{分组评估：}按风向扇区、可用率高低、正常工况/限电工况（若有标签）分别汇报，避免单一平均指标掩盖关键工况退化。
  \item \textbf{内部基线与消融：}至少包含 persistence / last-value baseline，以及当前 v1 已实现的 \texttt{graph\_off}、\texttt{wake\_off}、\texttt{no\_farm\_aux}、\texttt{no\_met\_aux}、\texttt{residual\_persistence} 等消融；同时可进一步比较去掉 PMU、将局部双状态 $(z,h)$ 合并为单状态等结构性变体。
\end{itemize}

\paragraph{强非图神经基线（固定）}
为严格评估动态图的净收益，v1 额外固定一个强的非图神经基线：\textbf{shared-weight TFT-no-graph}。该基线以每台机组为一个样本实例，输入同一套历史局部观测 $o_{i,t-L+1:t}$、静态属性 $s_i$、共享上下文 $c_{t-L+1:t}$ 与已知未来日历 $\mathrm{cal}_{t+1:t+H}$，但\textbf{不使用任何图边、邻机消息传递、局部/全局 latent state 分解或显式的 farm global state}。模型参数在所有机组间共享，训练窗口、数据切分、早停规则和主预测损失与主模型保持一致。这样在结果表中可以直接比较 ``shared covariate model'' 与 ``wake-aware dynamic graph model'' 之间的差异。

\paragraph{复现实验规范（固定）}
设全数据按时间排序后共有 $M$ 个统一时间步，默认固定连续切分边界为
\[
b_1=\lfloor 0.70M \rfloor,
\qquad
b_2=\lfloor 0.85M \rfloor,
\qquad
G=L+H=180\ \mathrm{steps}=30\ \mathrm{hours}.
\]
则训练集、验证集、测试集分别为
\[
\mathcal D_{train}=[1,b_1],\qquad
\mathcal D_{val}=[b_1+G+1,b_2],\qquad
\mathcal D_{test}=[b_2+G+1,M].
\]
这里预留的 $G=180$ 步 gap 用于切断 lookback 与 forecast horizon 对跨边界样本的感受野泄漏。所有缩放统计量、异常阈值统计和可学习初始化参数只能用 $\mathcal D_{train}$ 估计。

随机种子固定为 $\{3407,3408,3409\}$ 三个独立运行；主表汇报三次运行的均值 $\pm$ 标准差。Early stopping 指标固定为验证集上的 turbine-level MAE（对全部 horizon 均匀平均），patience 固定为 $10$ 个 epoch。Checkpoint 选择规则固定为：首先选验证 MAE 最低者；若差异小于 $10^{-4}$，则选验证 farm-level MAE 更低者；若仍并列，则取更早的 epoch。除非特别说明，所有对比模型都使用同一组切分、同一组随机种子和同一套 checkpoint 选择规则。

\section{未来可能的方向}

\subsection{搜索增强}

我能不能先定义一组小而关键的未来不确定因素，再训练一个模块在这些候选情景里做排序？首先，测试 generator 是否真的有“oracle 可选性”，也就是说加入扰动生成大量轨迹之后，是否真的有至少一条轨迹可以比不加扰动的时候得到更低的 loss。价值网络的任务是，针对某个时间步的大量候选，找到未来具有最高潜力的那几个扰动结果（也就是 Q Search）。不过到底是先 Q Search 直接走还是先 Rollout 再 V Search 剪枝得实验过才知道。

\begin{itemize}
  \item 能不能先定义一组低维且关键的未来不确定因素（最好带一定物理语义），再由 generator 在这些候选情景上生成大量未来分支？
  \item 首先需要检验 generator 是否真的具有“oracle 可选性”：也就是说，对同一个样本生成大量候选轨迹之后，是否至少存在一条轨迹能比 baseline（即不加候选情景/不加修正时）取得更低的未来 loss。若不存在，则问题主要在 generator，而不在 scorer。
  \item 如果 oracle 可选性存在，那么再训练一个排序模块来从大量候选中选出未来最有潜力的那几个分支。
  \item 若采用 Q-search，则训练的是一个 Q 网络 / 候选评分器，用来对“当前状态 + 下一步候选方向”直接打分，从而在每个时间步的大量候选中先筛出 top-$K$ 分支。
  \item 若采用 V-search，则先将候选方向 rollout 一步得到子状态，再用 V 网络对子状态进行评估和剪枝。
  \item 至于最终采用“先 Q-search 预筛，再 rollout”，还是“先 rollout，再 V-search 剪枝”，需要通过实验比较其效果与计算代价。
  \item 一个关键实验指标是 oracle best-of-$K$ 相对 baseline 的提升幅度，它决定了后续 scorer / search 是否值得做。
\end{itemize}

\subsection{结构扩展与更强图构造}

下列方向在概念上仍然有研究价值，但当前 v1 实现并未将其作为主干结构固定下来，因此不再放在前文主公式中，而统一留作后续扩展方向：

\begin{itemize}
  \item \textbf{独立的运行状态输入分支：}若后续数据侧能够稳定提供限功率、控制模式、检修/故障标签等信号，可重新引入单独的 $u^{ops}$ 分支，并比较“作为独立输入支路”与“仅通过事件摘要通道表达”两种方案的收益和实现复杂度。
  \item \textbf{稀疏物理邻域：}当前实现直接在全体有向边 $j\neq i$ 上做聚合。后续可进一步测试先用静态几何构造候选池、再按 wake gate 选取 top-$K$ 物理邻域的稀疏图方案，以降低计算量并强化局部物理归纳偏置。
  \item \textbf{learned residual edges：}纯几何 wake 图可能覆盖不了非局地相关性。未来可以在物理图之外增设少量 learned residual edges，用于补充通信异常、共模控制、场级限电等几何难以直接表达的相关结构。
  \item \textbf{更丰富的聚合统计：}当前实现的图聚合是门控加权平均。未来可比较更强的聚合器，例如并行保留 sum / mean / mass / max 多路统计，再由额外的聚合 MLP 生成最终消息，检验其是否能提升对上游负担强度和邻居异质性的表达能力。
  \item \textbf{更细粒度的 source summary 设计：}当前 source summary 由共享编码器直接学习得到。未来可测试显式子集投影、事件特征单独通路、或区分 update-source 与 transition-source 的更强参数化形式，以判断 source summary 是否值得从“隐式 learned summary”升级为“带结构约束的中间表示”。
\end{itemize}
